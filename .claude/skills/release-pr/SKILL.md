# 1. 事前確認

未コミットな変更がないことを確認してください

```bash
git status
```

# 2 mainブランチをチェックアウト

`git checkout main` で`main`ブランチに移ります

# 3. バージョンをどうするか下記選択肢からユーザーに尋ねて更新

`AskUserQuestion`ツールを使用して下記の選択肢からユーザーに尋ねてください。

- メジャーバージョンをインクリメントする（1.1.1 -> 2.0.0）
- マイナーバージョンをインクリメントする（1.1.1 -> 1.2.0）
- パッチバージョンをインクリメントする（1.1.1 -> 1.1.2）
- ユーザーから入力を受け付ける

指定された選択肢に従って、package.jsonのversionを書き換えたうえでコミットを行ってください。

コミットメッセージ: "バージョンを*.*.\*に更新"

# 5. PRの作成

ブランチをプッシュしたうえで、`production`ブランチに向けてPRを作成してください。

```bash
gh pr create --base production --title "<PRタイトル>" --body "<本文>"
```

PRタイトル: _._.\*のリリース準備
本文: 同上
